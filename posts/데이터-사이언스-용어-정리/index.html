데이터 사이언스 용어 정리

데이터 사이언스(Data Science)는 데이터를 수집하고 정제한 뒤, 통계적 분석과 머신러닝(Machine Learning)을 이용해 의미 있는 패턴을 찾고 의사결정에 활용하는 분야이다.

이 문서는 데이터 사이언스에서 자주 등장하는 핵심 용어를 개념, 사용 목적, 장점, 단점, 주의점 중심으로 정리한 문서이다.

전체 흐름

데이터 사이언스는 보통 다음 순서로 진행된다.

단계내용대표 개념
문제 정의어떤 질문을 풀 것인지 정한다목표 변수, 분석 목적
데이터 수집분석에 필요한 데이터를 모은다모집단, 표본, 관측치
데이터 이해데이터의 분포와 관계를 확인한다평균, 분산, 상관, 시각화
전처리결측치, 이상치, 스케일 차이를 처리한다Missing Value, Outlier, Scaling
모델링통계 모델 또는 머신러닝 모델을 학습한다Regression, Decision Tree, K-means
평가모델 성능과 해석 가능성을 확인한다Accuracy, F1-score, RMSE, MAE
활용예측, 분류, 의사결정, 자동화에 적용한다예측 모델, 추천, 탐지

1. 데이터의 기본 단위

모집단(Population)

모집단(Population)은 분석자가 알고 싶은 전체 대상이다.

예를 들어 전국 학생의 평균 키를 알고 싶다면 전국 학생 전체가 모집단이다. 통신 품질 분석에서는 전체 단말 로그, 전체 Call Drop 사례, 전체 RF 측정 결과 등이 모집단이 될 수 있다.

장점

장점설명
전체 특성 파악 가능모집단 전체를 분석하면 표본 오차가 없다
의사결정 신뢰도 높음전체 데이터 기반이므로 대표성 문제가 줄어든다

단점

단점설명
수집 비용 큼전체 데이터를 모으는 데 시간과 비용이 많이 든다
현실적으로 불가능한 경우 많음모든 사람, 모든 로그, 모든 장비 데이터를 수집하기 어렵다
데이터 관리 부담저장, 처리, 보안 비용이 커진다

표본(Sample)

표본(Sample)은 모집단에서 일부만 뽑은 데이터이다.

통계 분석에서는 대부분 모집단 전체를 알 수 없기 때문에 표본을 이용해 모집단의 특성을 추정한다.

장점

장점설명
비용 절감전체 데이터를 수집하지 않아도 된다
빠른 분석 가능적은 데이터로 모집단 특성을 추정할 수 있다
실험 가능A/B Test처럼 일부 집단을 대상으로 실험할 수 있다

단점

단점설명
표본 오차 발생표본이 모집단과 완전히 같을 수 없다
대표성 문제표본이 치우치면 잘못된 결론을 낼 수 있다
표본 수에 민감표본 수가 너무 작으면 추정이 불안정하다

모수(Parameter)

모수(Parameter)는 모집단의 특성을 나타내는 값이다.

예를 들어 모집단 평균, 모집단 분산, 모집단 비율이 모수이다.

통계량(Statistic)

통계량(Statistic)은 표본에서 계산한 값이다.

예를 들어 표본평균, 표본분산, 표본비율이 통계량이다.

구분의미예시
모수(Parameter)모집단의 실제 값모집단 평균, 모집단 분산
통계량(Statistic)표본에서 계산한 값표본평균, 표본분산

독립변수(Independent Variable)

독립변수(Independent Variable)는 결과를 설명하거나 예측하는 입력 변수이다.

머신러닝에서는 Feature 또는 Input Variable이라고도 한다.

예를 들어 집값 예측에서는 면적, 위치, 층수, 연식이 독립변수가 될 수 있다.

종속변수(Dependent Variable)

종속변수(Dependent Variable)는 예측하거나 설명하려는 결과 변수이다.

머신러닝에서는 Target, Label, Output Variable이라고도 한다.

예를 들어 집값 예측에서는 집값이 종속변수이다.

분석 상황독립변수종속변수
집값 예측면적, 위치, 연식집값
질병 예측나이, 혈압, 검사 수치질병 여부
RF 품질 예측RSRP, RSRQ, SINR, BandThroughput, Call Drop 여부

2. 기초 통계량

평균(Mean)

평균(Mean)은 모든 값을 더한 뒤 데이터 개수로 나눈 값이다.

장점

장점설명
계산이 쉽다전체 합을 개수로 나누면 된다
대표값으로 직관적데이터의 중심을 간단히 표현한다
통계 모델에서 자주 사용회귀, 정규분포, 가설검정의 기본 요소이다

단점

단점설명
이상치에 민감매우 큰 값이나 작은 값 하나가 평균을 크게 바꿀 수 있다
비대칭 분포에 부적절데이터가 한쪽으로 치우치면 대표성이 떨어진다

중앙값(Median)

중앙값(Median)은 데이터를 크기순으로 정렬했을 때 가운데 위치한 값이다.

장점

장점설명
이상치에 강함극단값이 있어도 크게 흔들리지 않는다
비대칭 분포에 유용소득, 집값처럼 한쪽 꼬리가 긴 데이터에 적합하다

단점

단점설명
전체 값을 모두 반영하지 않음데이터의 세부 크기 차이를 덜 반영한다
수학적 모델링에는 평균보다 덜 편리회귀나 분산 계산에서는 평균이 더 자주 사용된다

최빈값(Mode)

최빈값(Mode)은 가장 자주 등장하는 값이다.

범주형 데이터에서 대표값을 볼 때 유용하다.

장점

장점설명
범주형 데이터에 적합색상, 지역, 장비 종류처럼 평균 계산이 불가능한 값에 사용 가능하다
직관적가장 많이 나타난 값을 바로 보여준다

단점

단점설명
여러 개일 수 있음같은 빈도의 값이 여러 개면 해석이 애매하다
연속형 데이터에서는 덜 유용값이 다양하면 최빈값이 의미 없을 수 있다

분산(Variance)

분산(Variance)은 데이터가 평균으로부터 얼마나 떨어져 있는지를 제곱 기준으로 측정한 값이다.

장점

장점설명
퍼짐 정도를 수치화데이터 변동성을 표현할 수 있다
통계 이론에서 중요회귀, 검정, 분포 분석의 핵심 요소이다

단점

단점설명
단위 해석이 어렵다원래 단위의 제곱 형태이다
이상치에 민감편차를 제곱하므로 극단값 영향이 크다

표준편차(Standard Deviation)

표준편차(Standard Deviation)는 분산에 제곱근을 취한 값이다.

데이터가 평균 주변에 얼마나 퍼져 있는지 원래 단위로 해석할 수 있다.

장점

장점설명
원래 단위로 해석 가능점수, 전류, 전압, 속도 등 원래 단위와 같다
분포 폭을 직관적으로 표현값이 클수록 데이터가 넓게 퍼져 있다

단점

단점설명
이상치에 민감평균과 마찬가지로 극단값 영향을 받는다
비정규 분포에서는 해석 주의평균 ± 표준편차 해석이 항상 직관적이지 않다

3. 확률과 확률분포

확률(Probability)

확률(Probability)은 어떤 사건이 발생할 가능성을 0에서 1 사이 값으로 나타낸 것이다.

값의미
0절대 발생하지 않음
1반드시 발생함
0.5발생과 미발생 가능성이 같음

확률의 세 가지 공리

공리설명
비음수성모든 사건의 확률은 0 이상이다
전체확률 1표본공간 전체가 발생할 확률은 1이다
가산성서로 겹치지 않는 사건들의 합집합 확률은 각 확률의 합이다

조건부확률(Conditional Probability)

조건부확률(Conditional Probability)은 어떤 사건이 이미 발생했다는 조건에서 다른 사건이 발생할 확률이다.

표현은 P(AB)이고, B가 발생했을 때 A가 발생할 확률을 의미한다.

장점

장점설명
현실적인 의사결정 가능이미 관측된 정보에 따라 확률을 갱신할 수 있다
베이즈 정리의 기반검사 결과, 스팸 필터, 이상 탐지에 사용된다

단점

단점설명  
조건 방향을 헷갈리기 쉽다P(AB)와 P(BA)는 다르다
조건 데이터가 적으면 불안정특정 조건에 해당하는 표본 수가 적으면 확률 추정이 흔들린다  

독립 사건(Independent Events)

두 사건 A와 B가 서로 영향을 주지 않으면 독립이라고 한다.

독립이면 P(A and B) = P(A) × P(B)가 성립한다.

상호배반(Mutually Exclusive)

상호배반은 두 사건이 동시에 발생할 수 없는 경우이다.

A와 B가 상호배반이면 P(A and B) = 0이다.

주의할 점은 상호배반과 독립은 다른 개념이다. 두 사건이 상호배반이고 두 사건의 확률이 0이 아니라면 일반적으로 독립이 아니다.

4. 정규분포와 표준화

정규분포(Normal Distribution)

정규분포(Normal Distribution)는 평균을 중심으로 좌우 대칭인 종 모양의 연속확률분포이다.

측정 오차, 시험 점수, 신호 잡음처럼 여러 작은 요인의 합으로 만들어지는 값들이 정규분포에 가까운 경우가 많다.

핵심 특징

특징설명
좌우 대칭평균을 기준으로 왼쪽과 오른쪽 모양이 같다
평균, 중앙값, 최빈값 동일완전한 정규분포에서는 세 값이 같다
평균과 표준편차로 결정평균은 중심, 표준편차는 폭을 결정한다
68-95-99.7 법칙평균 ± 1, 2, 3 표준편차 범위에 약 68%, 95%, 99.7%가 포함된다

장점

장점설명
해석이 쉽다평균과 표준편차만으로 분포를 설명할 수 있다
통계 검정 기반z검정, t검정, 회귀의 오차 가정과 연결된다
자연 현상 근사에 유용측정 오차나 잡음 모델링에 자주 쓰인다

단점

단점설명
모든 데이터가 정규분포는 아님소득, 트래픽, 장애 횟수처럼 한쪽으로 치우친 데이터에는 부적합할 수 있다
이상치에 취약평균과 표준편차 기반이라 극단값에 영향을 받는다
꼬리 위험 과소평가 가능실제 데이터가 두꺼운 꼬리를 가지면 극단 사건을 낮게 평가할 수 있다

z-value 또는 z-score

z-value 또는 z-score는 특정 값이 평균으로부터 표준편차 몇 배만큼 떨어져 있는지를 나타내는 표준화 값이다.

z = (x - 평균) / 표준편차

z-value해석
0평균과 같다
양수평균보다 크다
음수평균보다 작다
절댓값이 큼평균에서 멀리 떨어져 있다

표준화(Standardization)

표준화(Standardization)는 데이터를 평균 0, 표준편차 1이 되도록 변환하는 방법이다.

장점

장점설명
변수 단위 차이 제거서로 다른 단위의 변수를 비교 가능하게 만든다
거리 기반 모델에 유리K-means, KNN(K-Nearest Neighbors), SVM(Support Vector Machine)에 중요하다
규제 모델에 유리Ridge, Lasso처럼 계수 크기를 제한하는 모델에서 필요하다

단점

단점설명
이상치 영향평균과 표준편차를 쓰므로 이상치에 민감하다
원래 단위 해석 어려움변환 후 값은 원래 물리 단위와 다르다

정규화(Normalization)

정규화(Normalization)는 데이터를 특정 범위로 변환하는 방법이다. 대표적으로 Min-Max Scaling은 값을 0에서 1 사이로 바꾼다.

장점

장점설명
범위가 명확모든 값이 정해진 구간 안에 들어온다
거리 기반 알고리즘에 유용변수 범위 차이가 거리 계산을 지배하는 문제를 줄인다

단점

단점설명
이상치에 매우 민감최솟값과 최댓값이 이상치이면 대부분 값이 좁은 범위에 몰린다
새로운 데이터 처리 주의학습 데이터 기준의 최솟값과 최댓값을 저장해서 적용해야 한다

중심극한정리(Central Limit Theorem)

중심극한정리(Central Limit Theorem)는 모집단 분포가 정규분포가 아니더라도, 표본 크기가 충분히 크면 표본평균의 분포가 정규분포에 가까워진다는 정리이다.

장점

장점설명
통계 추론의 기반신뢰구간과 가설검정이 가능해진다
비정규 모집단에도 적용 가능표본평균에 대해서는 정규 근사를 사용할 수 있다

단점

단점설명
표본 크기 조건 필요표본 수가 작으면 근사가 부정확할 수 있다
표본 독립성 필요관측치가 강하게 의존하면 적용이 어려울 수 있다
극단적 분포에서는 더 많은 표본 필요꼬리가 두꺼운 분포에서는 수렴이 느릴 수 있다

5. 상관과 공분산

공분산(Covariance)

공분산(Covariance)은 두 변수가 함께 움직이는 방향을 나타내는 값이다.

공분산 값해석
양수두 변수가 같은 방향으로 움직이는 경향
음수두 변수가 반대 방향으로 움직이는 경향
0 근처선형 관계가 약하거나 없을 수 있음

장점

장점설명
방향성 파악 가능두 변수가 같이 증가하는지 반대로 움직이는지 알 수 있다
회귀와 PCA의 기반변수 간 관계 구조를 분석하는 데 사용된다

단점

단점설명
크기 해석이 어렵다변수 단위에 따라 값이 달라진다
선형 관계 중심비선형 관계는 잘 표현하지 못한다

피어슨 상관계수(Pearson Correlation Coefficient)

피어슨 상관계수(Pearson Correlation Coefficient)는 두 연속형 변수 사이의 선형 관계 강도와 방향을 -1에서 1 사이 값으로 표현한다.

r 값해석
1완전한 양의 선형 관계
-1완전한 음의 선형 관계
0선형 관계가 거의 없음

장점

장점설명
표준화된 해석-1에서 1 사이 값이라 비교가 쉽다
선형 관계 파악에 유용회귀 전 탐색 분석에 적합하다
계산이 간단빠르게 변수 관계를 확인할 수 있다

단점

단점설명
비선형 관계에 약함곡선 관계가 강해도 r이 낮을 수 있다
이상치에 민감극단값 하나가 상관계수를 크게 바꿀 수 있다
인과관계 아님상관이 높아도 원인과 결과를 의미하지 않는다

스피어만 순위상관(Spearman Rank Correlation)

스피어만 순위상관(Spearman Rank Correlation)은 실제 값 대신 순위를 이용해 관계를 측정한다.

장점

장점설명
이상치에 덜 민감값 자체보다 순위를 사용한다
단조 관계 포착직선이 아니어도 한 방향으로 증가 또는 감소하면 관계를 잡을 수 있다
순위형 데이터에 적합등수, 만족도 등급 같은 데이터에 사용할 수 있다

단점

단점설명
실제 크기 차이를 잃음순위만 사용하므로 값의 간격 정보가 사라진다
복잡한 비단조 관계에는 부적합U자형 관계 같은 패턴은 잘 잡지 못한다

켄달 타우(Kendall Tau)

켄달 타우(Kendall Tau)는 두 변수의 순위 쌍이 얼마나 일관된 방향을 가지는지 측정하는 상관계수이다.

장점

장점설명
작은 표본에 비교적 안정적순위 쌍의 일치 여부를 기반으로 한다
순위형 데이터에 적합순위 비교가 중요한 문제에 유용하다

단점

단점설명
계산량이 상대적으로 큼모든 쌍 비교가 필요할 수 있다
해석이 피어슨보다 덜 직관적실무에서는 피어슨이나 스피어만보다 덜 익숙하다

상관과 인과(Correlation vs Causation)

상관이 높다고 해서 인과관계가 있다는 뜻은 아니다.

예를 들어 아이스크림 판매량과 익사 사고가 함께 증가한다고 해서 아이스크림이 익사 사고의 원인이라고 볼 수 없다. 둘 모두 여름이라는 숨은 변수의 영향을 받을 수 있다.

인과를 확인하려면 필요한 것

방법설명
무작위 실험처치와 대조군을 무작위로 나누어 원인 효과를 확인한다
A/B Test실제 서비스에서 두 조건을 비교한다
통제 변수 사용숨은 변수 영향을 줄이기 위해 모델에 포함한다
도메인 지식통계 결과를 현실적인 원인 구조와 함께 해석한다

6. 가설검정

가설검정(Hypothesis Testing)

가설검정(Hypothesis Testing)은 표본 데이터를 이용해 모집단에 대한 주장이나 가정을 통계적으로 판단하는 과정이다.

기본 흐름

단계내용
1귀무가설과 대립가설을 세운다
2유의 수준을 정한다
3검정통계량을 계산한다
4P-value 또는 임계값으로 판단한다
5귀무가설 기각 여부를 결정한다

귀무가설(H0, Null Hypothesis)

귀무가설은 차이가 없거나 효과가 없다는 기본 가설이다.

예시는 다음과 같다.

상황귀무가설
평균 비교두 집단 평균은 같다
회귀 분석회귀계수는 0이다
독립성 검정두 범주형 변수는 독립이다

대립가설(H1, Alternative Hypothesis)

대립가설은 차이가 있거나 효과가 있다는 가설이다. 보통 분석자가 확인하고 싶은 주장이다.

P-value

P-value는 귀무가설이 참이라고 가정했을 때, 현재 관측된 결과만큼 또는 그보다 더 극단적인 결과가 나올 확률이다.

해석 주의점

오해올바른 해석
P-value는 귀무가설이 참일 확률이다아니다. 귀무가설이 참이라고 가정했을 때 데이터가 나올 확률이다
P-value가 작으면 효과가 크다반드시 그렇지 않다. 표본 수가 크면 작은 효과도 유의할 수 있다
P-value가 크면 귀무가설이 참이다아니다. 기각할 근거가 부족하다는 뜻이다

유의 수준(Significance Level)

유의 수준(Significance Level)은 귀무가설이 참인데도 잘못 기각할 위험을 어디까지 허용할지 정한 기준이다.

보통 0.05를 많이 사용한다.

1종 오류(Type I Error)

1종 오류(Type I Error)는 귀무가설이 참인데 기각하는 오류이다.

즉, 실제 차이가 없는데 차이가 있다고 판단하는 경우이다.

2종 오류(Type II Error)

2종 오류(Type II Error)는 귀무가설이 거짓인데도 기각하지 못하는 오류이다.

즉, 실제 차이가 있는데 차이를 발견하지 못하는 경우이다.

검정력(Power)

검정력(Power)은 실제 효과가 있을 때 귀무가설을 올바르게 기각할 확률이다.

검정력이 높으면 실제 효과를 놓칠 가능성이 낮다.

신뢰구간(Confidence Interval)

신뢰구간(Confidence Interval)은 모집단 모수가 들어 있을 것으로 기대되는 범위를 표본 기반으로 제시하는 방법이다.

95% 신뢰구간은 같은 방식으로 표본 추출과 구간 계산을 반복했을 때, 그 구간들 중 약 95%가 참 모수를 포함한다는 의미이다.

장점

장점설명
효과 크기와 불확실성 확인단순히 유의한지만 보는 것보다 정보가 많다
실무 해석에 유용가능한 값의 범위를 확인할 수 있다

단점

단점설명
해석 오해 가능특정 계산된 구간에 참값이 있을 확률이 95%라는 뜻은 아니다
가정 위반에 민감표본 독립성, 분포 가정 등에 영향을 받는다

대표적인 검정 방법

z-검정(Z-test)

z-검정(Z-test)은 모집단 분산을 알거나 표본 수가 충분히 큰 경우 평균 검정에 사용한다.

장점단점
큰 표본에서 계산이 단순하다모집단 분산을 알아야 하는 경우가 많다
정규 근사와 연결된다작은 표본에서는 부적절할 수 있다

t-검정(T-test)

t-검정(T-test)은 모집단 분산을 모를 때 평균을 비교하는 검정이다.

종류사용 상황
일표본 t-검정(One-sample T-test)한 집단 평균이 특정 값과 다른지 확인
독립표본 t-검정(Independent-samples T-test)서로 독립인 두 집단 평균 비교
대응표본 t-검정(Paired T-test)같은 대상의 전후 차이 비교
장점단점
작은 표본에서도 사용 가능정규성 가정에 민감할 수 있다
평균 차이 검정에 직관적세 집단 이상 비교에는 부적절하다

카이제곱 검정(Chi-square Test)

카이제곱 검정(Chi-square Test)은 범주형 데이터의 독립성 또는 적합도를 검정할 때 사용한다.

장점단점
범주형 데이터 분석에 적합기대빈도가 너무 작으면 부정확할 수 있다
독립성 검정에 자주 사용관계의 강도나 방향을 직접 설명하지는 않는다

분산분석(ANOVA, Analysis of Variance)

분산분석(ANOVA, Analysis of Variance)은 세 집단 이상 평균 차이를 비교할 때 사용한다.

장점단점
여러 집단 평균을 한 번에 비교 가능어떤 집단끼리 다른지는 사후검정이 필요하다
1종 오류 증가를 줄임정규성, 등분산성 가정이 필요하다

비모수 검정(Non-parametric Test)

비모수 검정(Non-parametric Test)은 정규성 가정이 어렵거나 순위형 데이터일 때 사용하는 검정이다.

검정대응되는 상황
Mann-Whitney U Test독립된 두 집단 비교
Wilcoxon Signed-rank Test대응표본 비교
Kruskal-Wallis Test세 집단 이상 비교
장점단점
정규성 가정이 약하다평균 차이보다는 순위 차이를 본다
이상치에 비교적 강하다모수 검정보다 검정력이 낮을 수 있다

7. 회귀(Regression)

회귀의 개념

회귀(Regression)는 하나 이상의 독립변수를 사용해 종속변수를 설명하거나 예측하는 방법이다.

종속변수가 연속형이면 일반적으로 회귀 문제라고 한다.

단순선형회귀(Simple Linear Regression)

단순선형회귀(Simple Linear Regression)는 독립변수 1개와 연속형 종속변수 1개 사이의 관계를 직선으로 모델링한다.

예시는 다음과 같다.

독립변수종속변수
공부 시간시험 점수
광고비매출
RSRP(Reference Signal Received Power)Throughput

장점

장점설명
해석이 쉽다기울기 하나로 변수 영향력을 설명할 수 있다
계산이 빠르다모델 구조가 단순하다
기준 모델로 좋다복잡한 모델과 비교할 기본선이 된다

단점

단점설명
선형 관계만 표현곡선이나 복잡한 관계를 잘 표현하지 못한다
이상치에 민감극단값이 회귀선을 크게 흔들 수 있다
설명변수 1개만 사용현실 문제의 다양한 요인을 반영하기 어렵다

다중선형회귀(Multiple Linear Regression)

다중선형회귀(Multiple Linear Regression)는 여러 독립변수를 사용해 하나의 연속형 종속변수를 예측한다.

장점

장점설명
여러 요인 반영 가능현실 문제를 더 잘 설명할 수 있다
변수별 영향 해석 가능다른 변수를 고정했을 때 특정 변수의 효과를 볼 수 있다
통계적 검정 가능회귀계수의 유의성, 신뢰구간을 확인할 수 있다

단점

단점설명
다중공선성 문제독립변수끼리 강하게 상관되면 계수 해석이 불안정하다
선형성 가정 필요관계가 비선형이면 성능이 낮아질 수 있다
이상치와 영향점에 민감일부 데이터가 모델을 크게 바꿀 수 있다

회귀의 기본 가정

가정의미위반 시 문제
선형성(Linearity)독립변수와 종속변수 관계가 선형예측 편향 발생
독립성(Independence)관측치 또는 오차가 서로 독립표준오차와 검정 왜곡
등분산성(Homoscedasticity)오차 분산이 일정신뢰구간과 P-value 왜곡
정규성(Normality)오차가 정규분포에 가까움작은 표본에서 검정 부정확
다중공선성 약함독립변수 간 중복이 작음계수 해석 불안정

등분산성(Homoscedasticity)

등분산성(Homoscedasticity)은 독립변수 값이 달라져도 오차의 분산이 일정하다는 가정이다.

예를 들어 예측값이 작을 때는 오차가 작고, 예측값이 클 때는 오차가 커진다면 등분산성이 깨진 것이다.

왜 중요한가

이유설명
검정 신뢰성등분산성이 깨지면 회귀계수의 표준오차가 왜곡될 수 있다
신뢰구간 정확성구간 추정이 지나치게 좁거나 넓어질 수 있다
모델 진단잔차 패턴을 통해 모델이 놓친 구조를 찾을 수 있다

로지스틱 회귀(Logistic Regression)

로지스틱 회귀(Logistic Regression)는 종속변수가 이진 분류(Binary Classification)일 때 사용하는 모델이다.

예시는 다음과 같다.

문제종속변수
스팸 분류스팸 또는 정상
질병 예측질병 있음 또는 없음
구매 예측구매 또는 비구매

장점

장점설명
확률 출력결과를 0에서 1 사이 확률로 해석할 수 있다
해석 가능Odds Ratio를 통해 변수 영향력을 설명할 수 있다
빠르고 안정적기준 분류 모델로 많이 사용된다

단점

단점설명
선형 결정경계복잡한 비선형 분류에는 한계가 있다
다중공선성 영향독립변수 간 상관이 높으면 계수 해석이 어렵다
Feature Engineering 필요비선형 관계는 변환이나 상호작용 항이 필요할 수 있다

다항 회귀(Polynomial Regression)

다항 회귀(Polynomial Regression)는 입력 변수를 제곱, 세제곱 등으로 확장해 곡선 관계를 모델링한다.

장점

장점설명
비선형 패턴 표현곡선 형태의 관계를 표현할 수 있다
선형회귀 틀 유지계수 추정은 선형회귀 방식으로 가능하다

단점

단점설명
과적합 위험차수를 높이면 학습 데이터에 지나치게 맞을 수 있다
외삽에 취약관측 범위 밖 예측이 급격히 이상해질 수 있다
해석 어려움고차항이 많아지면 의미 해석이 복잡해진다

Ridge Regression

Ridge Regression은 L2 Regularization을 사용하는 회귀이다. 계수 크기를 전반적으로 줄여 과적합을 완화한다.

장점

장점설명
다중공선성 완화상관된 변수들이 많아도 계수 추정이 안정적이다
과적합 감소계수를 작게 만들어 모델 복잡도를 낮춘다
모든 변수를 유지변수 제거 없이 안정화할 수 있다

단점

단점설명
변수 선택 기능 약함계수를 0으로 만들지는 않는다
해석성 제한모든 변수가 남아 모델이 복잡할 수 있다
스케일링 필요변수 단위 차이가 페널티에 영향을 준다

Lasso Regression

Lasso Regression은 L1 Regularization을 사용하는 회귀이다. 중요하지 않은 변수의 계수를 0으로 만들 수 있다.

장점

장점설명
변수 선택 가능불필요한 변수의 계수를 0으로 만든다
해석성 향상남은 변수 중심으로 모델을 설명할 수 있다
고차원 데이터에 유용변수 수가 많은 데이터에서 효과적이다

단점

단점설명
상관 변수 처리 불안정비슷한 변수 중 하나만 선택할 수 있다
너무 강한 규제 위험중요한 변수까지 제거될 수 있다
스케일링 필요변수 단위에 따라 선택 결과가 달라질 수 있다

Elastic Net

Elastic Net은 L1과 L2 Regularization을 함께 사용하는 회귀이다.

장점

장점설명
Lasso와 Ridge 절충변수 선택과 계수 안정화를 함께 기대할 수 있다
상관 변수에 강함비슷한 변수들을 함께 유지할 가능성이 있다
고차원 데이터에 적합변수 수가 많고 상관이 큰 데이터에 유용하다

단점

단점설명
하이퍼파라미터 증가L1과 L2 비율을 조정해야 한다
해석 복잡단순 Lasso나 Ridge보다 설정과 해석이 복잡하다

포아송 회귀(Poisson Regression)

포아송 회귀(Poisson Regression)는 종속변수가 개수 데이터일 때 사용하는 GLM(Generalized Linear Model) 계열 모델이다.

예시는 방문자 수, 결함 개수, 사고 횟수, 특정 이벤트 발생 횟수이다.

장점

장점설명
개수 데이터에 적합음수가 될 수 없는 count 값을 모델링한다
발생률 분석 가능노출 시간이나 규모를 offset으로 반영할 수 있다
해석 가능변수 변화에 따른 발생률 변화를 볼 수 있다

단점

단점설명
평균과 분산 같다는 가정실제 데이터에서는 분산이 더 큰 과분산이 자주 발생한다
0이 많은 데이터에 약함Zero-inflated 데이터에는 별도 모델이 필요할 수 있다
독립성 가정 필요사건 간 의존성이 있으면 모델이 왜곡될 수 있다

GLM(Generalized Linear Model, 일반화선형모형)

GLM은 선형회귀를 확장한 모델이다. 종속변수가 정규분포가 아니어도 적절한 분포와 링크 함수를 선택해 모델링할 수 있다.

구성요소

구성요소의미
확률 성분(Random Component)종속변수가 따르는 분포
체계 성분(Systematic Component)독립변수들의 선형 결합
링크 함수(Link Function)종속변수 평균과 선형 결합을 연결하는 함수

대표 예시

모델종속변수분포링크 함수
선형회귀연속형정규분포Identity Link
로지스틱 회귀이진형베르누이 분포Logit Link
포아송 회귀개수형포아송분포Log Link

장점

장점설명
다양한 종속변수 처리연속형, 이진형, 개수형 데이터를 다룰 수 있다
해석 가능성회귀계수 기반 해석이 가능하다
통계적 추론 가능신뢰구간, 유의성 검정을 수행할 수 있다

단점

단점설명
분포와 링크 선택 필요잘못 선택하면 모델이 부정확하다
비선형 복잡 패턴에 한계복잡한 상호작용은 별도 feature가 필요하다
가정 진단 필요과분산, 독립성 위반 등을 확인해야 한다

8. 다중공선성

다중공선성(Multicollinearity)

다중공선성(Multicollinearity)은 독립변수들 사이에 강한 선형 관계가 존재하는 현상이다.

예를 들어 키와 팔 길이를 동시에 넣어 몸무게를 예측하면 두 변수는 서로 강하게 관련될 수 있다.

문제점

문제설명
계수 불안정데이터가 조금만 바뀌어도 회귀계수가 크게 변할 수 있다
부호 왜곡실제 기대와 반대 방향의 계수가 나올 수 있다
표준오차 증가유의하지 않은 변수처럼 보일 수 있다
해석 어려움어떤 변수가 실제 영향을 주는지 분리하기 어렵다

VIF(Variance Inflation Factor, 분산팽창계수)

VIF는 다중공선성 때문에 회귀계수 분산이 얼마나 커졌는지 보는 지표이다.

VIF일반적 해석
1공선성 거의 없음
5 이상공선성 의심
10 이상강한 공선성 가능성

완화 방법

방법설명장점단점
변수 제거중복 변수를 제거단순하고 해석 쉬움정보 손실 가능
변수 결합비슷한 변수를 평균이나 합성 지표로 결합의미 있는 지표 생성 가능결합 기준이 필요
PCA 사용상관된 변수를 주성분으로 축소공선성 완화 효과 큼원래 변수 해석 어려움
Ridge Regression 사용계수 크기를 줄여 안정화예측 안정성 향상변수 선택은 하지 않음
Lasso Regression 사용일부 계수를 0으로 만듦변수 선택 가능상관 변수 중 임의 선택 가능

9. 머신러닝 학습 방식

지도학습(Supervised Learning)

지도학습(Supervised Learning)은 정답 Label이 있는 데이터를 이용해 입력과 출력의 관계를 학습하는 방식이다.

대표 문제

문제설명예시
분류(Classification)범주를 예측스팸/정상, 양성/음성
회귀(Regression)연속값을 예측집값, 매출, Throughput

장점

장점설명
성능 평가가 명확정답이 있으므로 정확도나 오차를 계산할 수 있다
예측 문제에 적합새로운 데이터의 결과를 예측할 수 있다
실무 적용 범위 넓음분류, 회귀, 위험 예측에 많이 사용된다

단점

단점설명
Label 필요정답 데이터를 만드는 비용이 크다
Label 품질에 민감잘못된 Label은 모델 성능을 망친다
학습 데이터 범위 밖에 약함새로운 패턴에는 성능이 떨어질 수 있다

비지도학습(Unsupervised Learning)

비지도학습(Unsupervised Learning)은 정답 Label 없이 데이터 구조나 패턴을 찾는 방식이다.

대표 방법

방법설명예시
Clustering비슷한 데이터끼리 묶음고객 군집, 로그 패턴 군집
Dimensionality Reduction차원을 줄임PCA, 시각화
Association Rule같이 나타나는 패턴 탐색장바구니 분석

장점

장점설명
Label 없이 사용 가능정답 데이터가 없어도 분석할 수 있다
숨은 구조 발견사람이 몰랐던 군집이나 패턴을 찾을 수 있다
탐색 분석에 유용데이터 이해 단계에서 활용도가 높다

단점

단점설명
평가가 어렵다정답이 없어서 성능 판단이 애매하다
해석이 필요발견된 cluster나 pattern에 의미를 부여해야 한다
파라미터에 민감K값, 거리 기준, 최소 지지도 등에 따라 결과가 달라진다

준지도학습(Semi-supervised Learning)

준지도학습(Semi-supervised Learning)은 Label이 일부만 있는 데이터와 Label이 없는 데이터를 함께 사용하는 방식이다.

장점

장점설명
Label 비용 절감일부 Label만 있어도 학습 가능하다
대량의 미라벨 데이터 활용현실에서 흔한 비라벨 데이터를 사용할 수 있다

단점

단점설명
구현 복잡지도학습보다 설계가 어렵다
잘못된 pseudo label 위험잘못된 예측 Label이 성능을 악화시킬 수 있다

강화학습(Reinforcement Learning)

강화학습(Reinforcement Learning)은 Agent가 환경과 상호작용하면서 보상을 최대화하는 행동을 학습하는 방식이다.

장점

장점설명
순차적 의사결정에 적합게임, 로봇, 제어 문제에 강하다
장기 보상 최적화 가능지금 행동이 미래 결과에 미치는 영향을 학습한다

단점

단점설명
학습 비용 큼많은 시도와 환경 상호작용이 필요하다
안정성 문제잘못된 탐험이 위험하거나 비용이 클 수 있다
보상 설계 어려움보상을 잘못 설계하면 의도와 다른 행동을 학습한다

10. Decision Tree

Decision Tree의 개념

Decision Tree는 질문을 순서대로 던지면서 데이터를 분리하는 지도학습 모델이다.

분류(Classification)와 회귀(Regression) 모두에 사용할 수 있다.

예를 들어 날씨, 습도, 바람 정보를 보고 테니스를 칠지 여부를 예측하는 문제에서 Decision Tree는 다음과 같은 질문을 만든다.

질문의미
날씨가 Sunny인가첫 번째 분기 기준
습도가 High인가다음 분기 기준
바람이 Strong인가최종 판단 기준

Decision Tree는 지도학습인가

Decision Tree는 기본적으로 지도학습에 사용한다.

학습할 때 Feature와 정답 Label이 필요하기 때문이다.

구분예시
Feature날씨, 습도, 바람
LabelPlay 여부

불순도(Impurity)

불순도는 한 노드 안에 여러 클래스가 얼마나 섞여 있는지를 나타낸다.

상태불순도
한 클래스만 있음낮음
여러 클래스가 반반 섞임높음

Entropy

Entropy는 데이터가 얼마나 섞여 있는지를 측정하는 불순도 지표이다.

클래스가 완전히 하나이면 Entropy는 0이고, 클래스가 반반 섞이면 Entropy가 커진다.

Information Gain

Information Gain은 어떤 Feature로 데이터를 나누었을 때 불확실성이 얼마나 줄어드는지를 나타낸다.

Information Gain = 나누기 전 Entropy - 나눈 후 Entropy의 가중 평균

Decision Tree는 보통 Information Gain이 큰 Feature를 먼저 선택한다.

장점

장점설명
직관적질문과 분기로 설명할 수 있다
전처리 부담 적음스케일링이 거의 필요 없다
비선형 관계 표현 가능여러 조건 분기로 복잡한 경계를 만들 수 있다
Feature 중요도 확인 가능어떤 변수가 분기에 많이 쓰였는지 볼 수 있다

단점

단점설명
과적합 위험트리가 깊어지면 학습 데이터에 지나치게 맞는다
불안정성데이터가 조금 바뀌어도 트리 구조가 바뀔 수 있다
연속적 예측에 계단형 결과회귀에서는 예측값이 구간별 평균으로 나온다
편향 가능값 종류가 많은 Feature가 유리하게 선택될 수 있다

Gini Impurity

Gini Impurity는 임의로 선택한 데이터가 잘못 분류될 확률 관점의 불순도이다.

CART(Classification And Regression Tree) 계열에서 자주 사용한다.

Entropy와 Gini 비교

구분EntropyGini Impurity
계산 기준정보 이론 기반오분류 확률 기반
계산 복잡도로그 계산 필요상대적으로 단순
사용 알고리즘ID3, C4.5CART
결과 차이대체로 유사대체로 유사

Pruning

Pruning은 트리를 가지치기해서 과적합을 줄이는 방법이다.

방식설명

11. Bayes 정리

Bayes 정리의 개념

Bayes 정리는 어떤 증거가 관측되었을 때 원인의 확률을 갱신하는 방법이다.

즉, 기존 믿음에 새로운 정보를 반영해 사후확률을 계산한다.

Bayes 정리의 기본 형태는 다음과 같다.

P(AB) = P(BA) × P(A) / P(B)
용어의미
사전확률(Prior Probability)증거를 보기 전 확률
가능도(Likelihood)원인이 참일 때 증거가 나올 확률
증거확률(Evidence)증거가 발생할 전체 확률
사후확률(Posterior Probability)증거를 본 뒤 원인이 참일 확률

예시

질병 유병률이 1%, 질병이 있을 때 양성 확률이 99%, 질병이 없을 때 양성으로 잘못 나올 확률이 5%라고 하자.

검사 결과가 양성일 때 실제 질병이 있을 확률은 단순히 99%가 아니다.

전체 양성 중에는 실제 질병이 있어서 양성인 사람과, 질병은 없지만 위양성인 사람이 함께 포함되기 때문이다.

장점

장점설명
확률 업데이트 가능새로운 증거를 반영해 판단을 수정할 수 있다
직관적인 의사결정검사, 스팸 필터, 이상 탐지에 유용하다
사전정보 활용기존 도메인 지식을 모델에 반영할 수 있다

단점

단점설명

조건부확률 필요P(BA) 같은 값을 알아야 한다
계산 복잡성변수가 많아지면 계산이 복잡해진다

12. Clustering

Clustering의 개념

Clustering은 Label 없이 비슷한 데이터끼리 묶는 비지도학습 방법이다.

고객 세분화, 로그 패턴 분석, 이미지 그룹화, 이상 탐지 등에 사용된다.

K-means Clustering

K-means Clustering은 데이터를 K개의 cluster로 나누는 대표적인 알고리즘이다.

동작 순서는 다음과 같다.

단계내용
1K개의 centroid를 초기화한다
2각 데이터를 가장 가까운 centroid에 할당한다
3각 cluster의 평균으로 centroid를 갱신한다
4할당과 갱신을 반복한다
5centroid 변화가 작아지면 종료한다

장점

장점설명
간단하고 빠름대용량 데이터에도 비교적 잘 동작한다
해석이 쉬움중심점 기반으로 cluster를 설명할 수 있다
구현이 널리 제공대부분 라이브러리에서 지원한다

단점

단점설명
K를 미리 정해야 함cluster 개수를 사전에 알아야 한다
초기값에 민감centroid 초기화에 따라 결과가 달라진다
구형 cluster에 적합길쭉하거나 복잡한 모양의 cluster에는 약하다
이상치에 민감평균 기반이므로 outlier가 centroid를 흔든다
스케일링 필요변수 단위 차이가 거리 계산을 지배할 수 있다

K-means 초기화 방식

Random Initialization

Random Initialization은 데이터 중 K개를 무작위로 선택해 초기 centroid로 사용하는 방식이다.

장점단점
구현이 매우 쉽다실행할 때마다 결과가 달라질 수 있다
초기화 비용이 작다centroid가 한쪽에 몰릴 수 있다
작은 데이터에서는 충분히 유용local optimum에 빠질 수 있다

Forgy Method

Forgy Method는 전체 데이터에서 K개의 실제 sample을 랜덤 선택해 centroid로 사용하는 방식이다.

대부분의 기본 랜덤 초기화는 Forgy Method를 의미한다.

장점단점
실제 데이터 포인트를 중심으로 시작outlier가 선택될 수 있다
단순하고 빠름초기 centroid가 특정 영역에 몰릴 수 있다

Random Partition Method

Random Partition Method는 데이터를 먼저 무작위로 K개 cluster에 배정한 뒤, 각 cluster의 평균을 초기 centroid로 사용하는 방식이다.

장점단점
centroid가 데이터 전체 평균 근처에 분산될 수 있음실제 cluster 구조를 반영하지 못한다
빈 cluster 가능성을 줄일 수 있음처음부터 cluster가 섞여서 애매한 출발점이 될 수 있다

K-means++

K-means++는 첫 번째 centroid를 랜덤 선택한 뒤, 이후 centroid는 기존 centroid와 멀리 떨어진 점이 선택될 확률을 높이는 방식이다.

장점단점

수렴 속도 향상완전한 최적해를 보장하지는 않는다
결과 안정성 증가outlier가 멀리 있으면 선택될 가능성이 있다

Maximin Initialization

Maximin Initialization은 기존 centroid와 가장 멀리 떨어진 점을 다음 centroid로 선택한다.

장점단점

Multiple Random Starts는 여러 초기값으로 K-means를 반복 실행하고, inertia 또는 WCSS(Within-Cluster Sum of Squares)가 가장 작은 결과를 선택한다.

장점단점
나쁜 초기값 위험 감소계산 시간이 증가한다

14. Association Rule

연관규칙(Association Rule)의 개념

연관규칙(Association Rule)은 데이터에서 함께 자주 등장하는 item 조합을 찾는 비지도학습 방법이다.

대표적으로 장바구니 분석에 사용된다.

규칙 형태는 다음과 같다.

A -> B

이 의미는 A가 등장했을 때 B도 함께 등장하는 경향이 있다는 뜻이다.

지지도(Support)

지지도(Support)는 특정 itemset이 전체 transaction 중 얼마나 자주 등장하는지 나타낸다.

Support(A -> B) = A와 B가 모두 포함된 transaction 수 / 전체 transaction 수

핵심 주의점

구분설명
방향성Support(A -> B)와 Support(B -> A)는 같다
기준전체 transaction 수로 나눈다
의미규칙이 데이터 전체에서 얼마나 흔한지 나타낸다

신뢰도(Confidence)

신뢰도(Confidence)는 A가 등장한 transaction 중 B도 등장한 비율이다.

Confidence(A -> B) = A와 B가 모두 포함된 transaction 수 / A가 포함된 transaction 수

향상도(Lift)

향상도(Lift)는 A와 B가 독립적으로 등장하는 경우보다 얼마나 더 강하게 함께 나타나는지 나타낸다.

Lift(A -> B) = Confidence(A -> B) / Support(B)

Lift 값해석
1보다 큼A와 B가 함께 나타나는 경향이 강함
1독립에 가까움
1보다 작음A가 있을 때 B가 오히려 덜 나타남

Apriori Algorithm

Apriori Algorithm은 빈번한 itemset을 찾는 대표 알고리즘이다.

핵심 원리는 어떤 itemset이 빈번하려면 그 부분집합도 모두 빈번해야 한다는 것이다.

장점

장점설명
해석이 쉽다규칙 형태로 결과를 설명할 수 있다
불필요한 후보 감소Apriori property로 탐색 공간을 줄인다
장바구니 분석에 적합상품 조합 분석에 많이 사용된다

단점

단점설명
후보 생성 비용 큼item 수가 많으면 후보 itemset이 폭증한다
희소 데이터에 어려움transaction이 sparse하면 의미 있는 규칙이 적을 수 있다
지지도 기준 민감최소 지지도를 어떻게 정하느냐에 따라 결과가 크게 달라진다

FP-Growth

FP-Growth(Frequent Pattern Growth)는 후보 itemset을 직접 많이 생성하지 않고 FP-Tree를 사용해 빈번 패턴을 찾는 방법이다.

장점

장점설명
Apriori보다 빠른 경우 많음후보 생성 비용을 줄인다
대규모 transaction에 유리반복적인 database scan을 줄일 수 있다

단점

단점설명
구현이 복잡FP-Tree 구조를 이해해야 한다
메모리 사용 가능데이터 구조가 커질 수 있다
해석은 여전히 규칙 필터링 필요너무 많은 규칙이 나올 수 있다

15. 모델 평가 지표

혼동행렬(Confusion Matrix)

혼동행렬은 분류 모델의 예측 결과를 실제 값과 비교해 정리한 표이다.

구분의미
TP(True Positive, 진양성)실제 양성을 양성으로 맞춘 경우
TN(True Negative, 진음성)실제 음성을 음성으로 맞춘 경우
FP(False Positive, 위양성)실제 음성을 양성으로 잘못 예측한 경우
FN(False Negative, 위음성)실제 양성을 음성으로 잘못 예측한 경우

정확도(Accuracy)

정확도는 전체 데이터 중 맞게 예측한 비율이다.

Accuracy = (TP + TN) / (TP + TN + FP + FN)

장점

장점설명
직관적전체 중 얼마나 맞췄는지 바로 알 수 있다
균형 데이터에 유용클래스 비율이 비슷하면 해석이 쉽다

단점

단점설명
불균형 데이터에 취약양성이 1%인 문제에서 전부 음성이라 해도 99% 정확도가 나올 수 있다
오류 종류 구분 불가FP와 FN의 비용 차이를 반영하지 못한다

정밀도(Precision)

정밀도(Precision)는 모델이 양성이라고 예측한 것 중 실제 양성의 비율이다.

Precision = TP / (TP + FP)

장점

장점설명
위양성 관리에 유용잘못된 양성 경고를 줄이는 데 중요하다
추천, 스팸 필터에 적합양성 예측의 신뢰도를 본다

단점

단점설명
위음성 반영 약함실제 양성을 얼마나 놓쳤는지는 알기 어렵다
단독 사용 위험정밀도만 높이고 많은 양성을 놓칠 수 있다

재현율(Recall) 또는 민감도(Sensitivity)

재현율(Recall)은 실제 양성 중 모델이 양성으로 맞춘 비율이다.

Recall = TP / (TP + FN)

장점

장점설명
위음성 관리에 유용실제 양성을 놓치지 않는 것이 중요할 때 사용한다
의료, 불량 탐지에 적합놓치면 위험한 문제에서 중요하다

단점

단점설명
위양성 증가 가능재현율을 높이면 불필요한 양성 예측이 많아질 수 있다
단독 사용 위험모든 데이터를 양성으로 예측하면 재현율은 높아질 수 있다

특이도(Specificity)

특이도(Specificity)는 실제 음성 중 모델이 음성으로 맞춘 비율이다.

Specificity = TN / (TN + FP)

장점

장점설명
위양성 감소 관점실제 음성을 잘 보호하는지 확인할 수 있다
선별 검사 보완민감도와 함께 보면 진단 성능을 균형 있게 평가한다

단점

단점설명
양성 탐지 능력은 별도 확인 필요특이도만 높아도 양성을 놓칠 수 있다
클래스 불균형에 주의음성이 매우 많으면 높게 보일 수 있다

F1-score

F1-score는 정밀도와 재현율의 조화평균이다.

F1-score = 2 × Precision × Recall / (Precision + Recall)

장점

장점설명
정밀도와 재현율 균형둘 중 하나만 높은 모델을 덜 좋게 평가한다
불균형 데이터에 유용Accuracy보다 현실적인 평가가 가능하다

단점

단점설명
TN을 반영하지 않음음성을 잘 맞추는 능력은 직접 반영하지 않는다
비용 차이 반영 어려움FP와 FN 중 어떤 오류가 더 중요한지 별도 고려해야 한다

ROC Curve와 AUC

ROC(Receiver Operating Characteristic) Curve는 threshold 변화에 따른 TPR(True Positive Rate)과 FPR(False Positive Rate)의 관계를 나타낸다.

AUC(Area Under the Curve)는 ROC Curve 아래 면적이다.

장점

장점설명
threshold 독립 평가특정 threshold 하나에 의존하지 않는다
모델 구분 능력 평가양성과 음성을 얼마나 잘 구분하는지 본다

단점

단점설명
불균형 데이터에서 과대평가 가능음성이 매우 많으면 좋아 보일 수 있다
실제 운영 threshold 선택은 별도AUC가 높아도 어떤 기준값을 쓸지는 따로 정해야 한다

PR Curve와 Average Precision

PR Curve(Precision-Recall Curve)는 threshold 변화에 따른 Precision과 Recall의 관계를 나타낸다.

데이터 불균형이 심한 양성 탐지 문제에서 ROC보다 유용한 경우가 많다.

장점

장점설명
희귀 양성 문제에 적합양성이 적은 fraud, 질병, 장애 탐지에 유용하다
양성 예측 품질 확인Precision과 Recall trade-off를 직접 볼 수 있다

단점

단점설명
해석이 ROC보다 낯설 수 있음기준선이 양성 비율에 따라 달라진다
threshold 선택 필요운영 목적에 맞는 지점을 골라야 한다

회귀 평가 지표

MAE(Mean Absolute Error)

MAE는 실제값과 예측값 차이의 절댓값 평균이다.

장점단점
원래 단위로 해석이 쉽다큰 오차에 대한 패널티가 상대적으로 작다
이상치에 RMSE보다 덜 민감미분이 매끄럽지 않은 지점이 있다

MSE(Mean Squared Error)

MSE는 오차 제곱의 평균이다.

장점단점
큰 오차를 강하게 벌점단위가 제곱이라 해석이 어렵다
최적화에 편리이상치에 민감하다

RMSE(Root Mean Squared Error)

RMSE는 MSE에 제곱근을 취한 값이다.

장점단점
원래 단위로 해석 가능큰 오차에 민감하다
큰 예측 실패를 중요하게 반영이상치가 많으면 모델 비교가 왜곡될 수 있다

R-squared

R-squared는 종속변수 변동 중 모델이 설명하는 비율이다.

장점단점
설명력 해석 가능변수를 추가하면 보통 증가한다
회귀 모델 비교에 자주 사용예측 성능을 보장하지 않는다

Adjusted R-squared

Adjusted R-squared는 변수 수를 고려해 보정한 결정계수이다.

장점단점
불필요한 변수 추가를 벌점모든 예측 문제에 적합한 지표는 아니다
변수 수가 다른 회귀 모델 비교에 유용비선형 모델 평가에는 제한적이다

16. 데이터 전처리

결측치(Missing Value)

결측치는 값이 비어 있는 데이터이다.

처리 방법

방법장점단점
행 삭제간단함데이터 손실 큼
평균 대체구현 쉬움분산 감소, 이상치 영향
중앙값 대체이상치에 강함변수 간 관계 반영 약함
최빈값 대체범주형에 사용 가능데이터 다양성 감소
모델 기반 대체다른 변수 정보 활용구현 복잡, 누수 위험
결측 여부 Feature 추가결측 자체의 의미 반영Feature 증가

이상치(Outlier)

이상치는 대부분 데이터와 크게 다른 관측값이다.

처리 방법

방법장점단점
제거오류 데이터 제거 가능중요한 희귀 현상 손실 가능
Winsorization극단값 영향을 줄임원본 데이터 왜곡
Robust Scaling중앙값과 IQR 사용모든 모델 문제를 해결하지는 않음
별도 분석이상 탐지에 유용분석 복잡도 증가

Encoding

Encoding은 범주형 변수를 숫자로 변환하는 과정이다.

방식설명장점단점
Label Encoding범주를 정수로 변환간단함순서가 없는 범주에 가짜 순서 부여
One-hot Encoding범주마다 0/1 변수 생성순서 왜곡 없음범주 수 많으면 차원 증가
Target Encoding범주별 target 평균 사용고카디널리티에 유용데이터 누수 위험
Frequency Encoding범주 빈도로 변환간단하고 차원 증가 없음target과 직접 관계 반영 약함

데이터 누수(Data Leakage)

데이터 누수는 학습 시점에 알 수 없어야 하는 정보가 모델 학습에 들어가는 문제이다.

예를 들어 테스트 데이터 전체를 기준으로 스케일링한 뒤 train/test를 나누면 테스트 정보가 학습 과정에 들어간다.

방지 방법

방법설명
train/test 먼저 분리전처리 기준은 train에서만 학습한다
Cross Validation 내부에서 전처리fold마다 전처리를 따로 fit한다
미래 정보 제거예측 시점 이후 정보를 feature에 넣지 않는다
Target Encoding 주의검증 fold의 target 정보를 쓰지 않는다

17. 모델 일반화와 검증

과적합(Overfitting)

과적합은 모델이 학습 데이터에 지나치게 맞춰져 새로운 데이터 성능이 떨어지는 상태이다.

원인

원인설명
모델이 너무 복잡파라미터가 많거나 트리가 너무 깊다
데이터가 적음우연한 패턴까지 학습한다
노이즈가 많음잡음을 실제 패턴으로 착각한다

완화 방법

방법설명
Regularization계수나 모델 복잡도 제한
Cross Validation여러 검증 분할로 성능 확인
PruningTree의 불필요한 가지 제거
Early Stopping검증 성능이 나빠지기 전에 학습 중단
데이터 증가더 다양한 학습 데이터를 확보

과소적합(Underfitting)

과소적합은 모델이 너무 단순해서 학습 데이터의 패턴도 충분히 설명하지 못하는 상태이다.

완화 방법

방법설명
더 복잡한 모델 사용선형모델에서 Tree, Boosting 등으로 변경
Feature 추가중요한 설명변수를 추가
비선형 변환다항항, 로그 변환, interaction 추가
규제 완화너무 강한 regularization을 줄임

편향-분산 트레이드오프(Bias-Variance Tradeoff)

편향(Bias)은 모델이 실제 패턴을 단순화해서 생기는 오차이고, 분산(Variance)은 학습 데이터 변화에 모델이 민감하게 흔들리는 정도이다.

상태특징
높은 Bias과소적합 가능성
높은 Variance과적합 가능성
적절한 균형일반화 성능 우수

Train, Validation, Test Split

데이터용도
Training Data모델 학습
Validation Data모델 선택, hyperparameter 조정
Test Data최종 성능 평가

Cross Validation

Cross Validation은 데이터를 여러 fold로 나누어 학습과 검증을 반복하는 방법이다.

대표 방식

방식설명
K-fold Cross Validation데이터를 K개 fold로 나누어 반복 평가
Stratified K-fold클래스 비율을 유지하며 fold 분할
Time Series Split시간 순서를 지키며 검증
Leave-One-Out하나만 검증 데이터로 두고 반복

장점

장점설명
성능 평가 안정적하나의 train/test split에 덜 의존한다
데이터 효율적 사용모든 데이터가 학습과 검증에 참여한다

단점

단점설명
계산 비용 증가fold 수만큼 학습을 반복한다
시간 데이터에는 주의랜덤 분할하면 미래 정보가 과거 학습에 들어갈 수 있다

20. 개념 연결 흐름

데이터 사이언스에서는 먼저 모집단과 표본을 구분하고, 평균, 분산, 표준편차로 데이터의 중심과 퍼짐을 본다. 데이터 분포가 정규분포에 가까운지 확인하면 z-score, 신뢰구간, 가설검정 같은 통계 추론을 이해하기 쉬워진다.

변수 사이 관계를 볼 때는 공분산과 상관계수를 사용하지만, 상관이 인과를 의미하지 않는다는 점을 주의해야 한다. 예측이 목적이면 회귀, 분류, Decision Tree 같은 지도학습 모델을 사용한다. 정답 Label이 없고 데이터 구조를 찾는 것이 목적이면 K-means, DBSCAN, PCA, Association Rule 같은 비지도학습 방법을 사용할 수 있다.

모델링 후에는 반드시 평가 지표를 문제 목적에 맞게 선택해야 한다. 양성을 놓치면 위험한 문제에서는 Recall 또는 Sensitivity가 중요하고, 잘못된 양성 경고를 줄이는 것이 중요하면 Precision 또는 Specificity가 중요하다. 회귀 문제에서는 MAE, RMSE, R-squared를 함께 보고, 과적합과 데이터 누수 여부를 항상 점검해야 한다.
