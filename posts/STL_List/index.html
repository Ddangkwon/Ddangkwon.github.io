C++ STL List 정리

1. STL List란

STL(Standard Template Library)의 List는 이중 연결 리스트(Doubly Linked List) 기반의 컨테이너이다.

각 원소는 메모리상에 연속적으로 붙어 있지 않고, 각 노드가 다음 노드와 이전 노드를 가리키는 방식으로 연결된다.

즉, Vector처럼 배열 기반이 아니라 노드 기반 자료구조라고 보면 된다.

2. 기본 특징

  • 중간 삽입과 삭제가 빠르다.
  • 맨 앞과 맨 뒤 삽입, 삭제가 빠르다.
  • 임의 접근(Random Access)이 느리다.
  • 메모리가 연속적이지 않다.
  • 각 원소마다 포인터 정보가 추가로 필요하므로 메모리 오버헤드가 있다.

3. 내부 구조 개념

List는 보통 다음과 같은 구조를 가진다.

Node 1 <-> Node 2 <-> Node 3 <-> Node 4

각 노드는 다음 정보를 가진다.

  • 현재 데이터
  • 이전 노드를 가리키는 포인터
  • 다음 노드를 가리키는 포인터

예를 들어 값이 10, 20, 30인 List는 개념적으로 다음과 같다.

NULL <- [10] <-> [20] <-> [30] -> NULL

이 구조 때문에 특정 위치를 알고 있다면 그 위치 앞뒤 삽입, 삭제는 매우 빠르다.

5. 주요 멤버 함수

5-1. push_back

맨 뒤에 원소를 추가한다.

1
2
3

    list<int> lst;

    lst.push_back(20);

시간복잡도는 보통 O(1) 이다.

맨 앞에 원소를 추가한다.

1

    lst.push_front(5);

결과

1

5 10 20

시간복잡도는 보통 O(1) 이다.

5-3. pop_back

맨 뒤 원소를 삭제한다.

1

    lst.pop_back();

5-4. pop_front

맨 앞 원소를 삭제한다.

1

    lst.pop_front();

맨 앞과 맨 뒤 원소에 접근한다.

1
2

    cout << lst.front() << "\n";
    cout << lst.back() << "\n";

5-7. clear

모든 원소를 삭제한다.

1

    lst.clear();

5-8. insert

특정 iterator 위치 앞에 원소를 삽입한다.

1
2

4

    list<int> lst = {10, 20, 30};

    auto it = lst.begin();
    ++it;

    lst.insert(it, 15);

결과

1

중요한 점은 List는 인덱스로 접근하지 않고 iterator 기반으로 위치를 지정한다는 점이다.

5-9. erase

특정 위치의 원소를 삭제한다.

1
2
3
4

    auto it = lst.begin();
    ++it;

    lst.erase(it);

결과

1

10 20 30

범위 삭제도 가능하다.

1
2
3
4
5
6

    auto it1 = lst.begin();
    auto it2 = lst.begin();
    ++it2;
    ++it2;

    lst.erase(it1, it2);

이 경우 시작 iterator는 포함되고 끝 iterator는 포함되지 않는다.

5-10. remove

값이 같은 모든 원소를 삭제한다.

1
2

    list<int> lst = {1, 2, 3, 2, 4, 2};
    lst.remove(2);

결과

1

1 3 4

5-11. sort

List 내부 원소를 정렬한다.

1
2

    list<int> lst = {4, 2, 5, 1, 3};
    lst.sort();

결과

1

1 2 3 4 5

내림차순 정렬도 가능하다.

1

    lst.sort(greater<int>());

5-12. reverse

원소 순서를 뒤집는다.

1

    lst.reverse();

5-13. unique

연속해서 중복된 원소를 하나만 남기고 제거한다.

1
2

    list<int> lst = {1, 1, 2, 2, 2, 3, 1, 1};
    lst.unique();

결과

1 2 3 1

중요한 점은 연속 중복만 제거한다는 점이다. 전체 중복 제거가 아니라 붙어 있는 중복만 제거한다.

5-14. merge

정렬된 두 List를 하나로 합친다.

1
2
3
4

    list<int> a = {1, 3, 5};
    list<int> b = {2, 4, 6};

    a.merge(b);

결과

a 는 1 2 3 4 5 6 b 는 빈 리스트가 된다.

주의할 점은 보통 두 List가 이미 정렬되어 있어야 의미 있는 결과가 나온다는 점이다.

5-15. splice

다른 List의 노드를 복사하지 않고 현재 List로 옮긴다.

1
2
3
4
5
6
7

    list<int> a = {1, 2, 3};
    list<int> b = {10, 20, 30};

    auto it = a.begin();
    ++it;

    a.splice(it, b);

결과

a 는 1 10 20 30 2 3 b 는 빈 리스트가 된다.

splice는 List의 강력한 장점 중 하나이다. 노드 연결만 바꾸므로 매우 효율적이다.

6. iterator

List는 Random Access Iterator가 아니라 Bidirectional Iterator(양방향 반복자) 를 제공한다.

즉 가능한 연산은 다음과 같다.

  • 앞으로 이동
  • 뒤로 이동

가능 예시

1
2
3

    auto it = lst.begin();
    ++it;
    --it;

불가능 예시

1

    it = it + 3;

Vector에서는 가능하지만 List에서는 불가능하다.

즉 List는 인덱스 기반 접근이 아니라 한 칸씩 이동하는 구조이다.

7. List와 Vector 비교

항목ListVector
내부 구조이중 연결 리스트동적 배열
메모리 배치비연속연속
임의 접근느림빠름
맨 뒤 삽입빠름빠름
맨 앞 삽입빠름느림
중간 삽입, 삭제위치를 알고 있으면 빠름느림
캐시 효율낮음높음
정렬가능가능

핵심 차이

  • Vector는 배열 기반이라 접근이 빠르다.
  • List는 삽입과 삭제가 강하다.
  • 일반적으로 단순 순회, 인덱스 접근, 캐시 효율은 Vector가 더 좋다.
  • 노드 이동 기반 작업은 List가 유리할 수 있다.

8. 시간복잡도 정리

연산시간복잡도
push_backO(1)
push_frontO(1)
pop_backO(1)
pop_frontO(1)
front, backO(1)
insert위치를 알고 있으면 O(1)
erase위치를 알고 있으면 O(1)
find 성격 탐색O(N)
특정 위치까지 이동O(N)
sortO(N log N)
size구현체에 따라 O(1)

주의할 점은 insert와 erase가 빠르다고 해도 그 위치까지 iterator를 이동하는 데 O(N)이 걸릴 수 있다는 점이다.

즉 실제 사용에서는 다음처럼 생각해야 한다.

  • 위치를 이미 알고 있다면 빠름
  • 위치를 찾는 과정까지 포함하면 느릴 수 있음

9. 자주 하는 실수

9-1. 인덱스로 접근하려고 하는 실수

1
2
3
4

    list<int> lst = {1, 2, 3};

    // 불가능한 접근
    // cout << lst[1] << "\n";

List는 배열이 아니므로 인덱스 연산자를 지원하지 않는다.

9-2. iterator 덧셈을 하려는 실수

1
2
3
4

    auto it = lst.begin();

    // 불가능
    // it = it + 2;

List iterator는 Random Access Iterator가 아니다.

9-3. unique를 전체 중복 제거로 오해하는 실수

1
2

    list<int> lst = {1, 2, 1, 2, 1};
    lst.unique();

결과는 그대로 1 2 1 2 1 이다. 연속 중복이 아니기 때문이다.

전체 중복 제거를 원하면 보통 먼저 정렬한 뒤 unique를 사용한다.

1
2

    lst.sort();
    lst.unique();

9-4. Vector보다 무조건 빠를 것이라고 오해하는 실수

List는 삽입, 삭제가 빠르지만 실제 프로그램에서는 다음 이유로 Vector가 더 빠른 경우가 매우 많다.

  • 캐시 효율이 좋음
  • 메모리 지역성(locality)이 좋음
  • 노드 포인터 추적 비용이 없음

즉 무조건 List를 고르는 것은 좋지 않다.

10. 언제 List를 쓰면 좋은가

다음 상황에서 고려할 수 있다.

  • 중간 삽입, 삭제가 매우 자주 발생한다.
  • 이미 해당 위치를 iterator로 알고 있다.
  • 노드 단위 이동이 중요하다.
  • splice, merge 같은 List 전용 기능이 유용하다.

반대로 다음 상황에서는 대체로 Vector가 더 유리하다.

  • 인덱스로 자주 접근해야 한다.
  • 단순 순회 성능이 중요하다.
  • 메모리 효율과 캐시 효율이 중요하다.
  • 삽입, 삭제보다 조회가 훨씬 많다.

15. 실무적, 코딩테스트 관점 정리

실무 관점

실제 프로그램에서는 List보다 Vector가 더 자주 쓰인다. 이유는 캐시 효율과 순차 접근 성능 때문이다.

코딩테스트 관점

코딩테스트에서는 다음 상황에서 List를 고려할 수 있다.

  • 중간 삽입, 삭제가 많음
  • iterator 기반 조작이 중요함
  • 양방향 연결 구조를 직접 구현하기 싫음

하지만 코딩테스트에서도 무조건 List를 쓰기보다는 문제 특성을 먼저 보고 선택해야 한다.

16. 한 줄 요약

List는 이중 연결 리스트 기반 컨테이너이므로 중간 삽입과 삭제에 강하지만, 인덱스 접근과 순차 성능 면에서는 Vector보다 불리할 수 있다.

즉 핵심은 다음이다.

  • 삽입, 삭제 중심이면 List 고려
  • 조회, 순회 중심이면 Vector 우선 고려

17. 암기 포인트

  • List는 이중 연결 리스트 기반이다.
  • 인덱스 접근이 불가능하다.
  • iterator는 양방향 이동만 가능하다.
  • 중간 삽입, 삭제는 위치를 알고 있으면 빠르다.
  • sort, merge, splice 같은 List 전용 강점이 있다.
  • 실제 성능은 Vector가 더 좋은 경우가 많다.
