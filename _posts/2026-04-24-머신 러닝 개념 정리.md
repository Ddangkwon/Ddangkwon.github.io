# 1. Decision Tree와 Supervised Learning

## Decision Tree는 Supervised Learning인가?

Decision Tree는 기본적으로 **Supervised Learning**에 사용하는 알고리즘이다.

Supervised Learning은 학습 데이터에 정답 label이 포함되어 있는 학습 방식이다.

예를 들어 다음과 같은 데이터가 있다고 하자.

| Weather | Humidity | Wind | Play |
|---|---|---|---|
| Sunny | High | Weak | No |
| Sunny | Normal | Weak | Yes |
| Rainy | High | Strong | No |
| Cloudy | High | Weak | Yes |

여기서 구분은 다음과 같다.

| 구분 | 예시 | 의미 |
|---|---|---|
| Feature, 독립변수 | Weather, Humidity, Wind | 예측에 사용하는 입력값 |
| Label, Target, 종속변수 | Play | 맞춰야 하는 정답 |

Decision Tree는 Feature를 기준으로 데이터를 나누면서 Label을 잘 예측하는 규칙을 학습한다.

## Decision Tree가 사용되는 문제 유형

Decision Tree는 크게 두 가지 문제에 사용할 수 있다.

| 문제 유형 | 설명 | 예시 |
|---|---|---|
| Classification | 범주형 값을 예측 | 스팸 여부, 합격 여부, 질병 여부 |
| Regression | 연속형 숫자 값을 예측 | 집값, 매출, 온도, 전류값 |

## Classification 예시

| 공부시간 | 출석률 | 결과 |
|---:|---:|---|
| 8 | 95% | Pass |
| 2 | 60% | Fail |
| 6 | 80% | Pass |

결과가 Pass 또는 Fail처럼 범주형 값이면 Classification 문제이다.

## Regression 예시

| 면적 | 위치점수 | 집값 |
|---:|---:|---:|
| 84 | 8 | 7억 |
| 59 | 6 | 5억 |
| 110 | 9 | 10억 |

집값처럼 숫자를 예측하면 Regression 문제이다.

## Supervised Learning과 Unsupervised Learning 비교

| 구분 | 정답 label 필요 여부 | 대표 알고리즘 |
|---|---|---|
| Supervised Learning | 필요 | Decision Tree, Linear Regression, Logistic Regression, SVM |
| Unsupervised Learning | 필요 없음 | K-means Clustering, PCA, Association Rule |

핵심은 다음이다.

Decision Tree는 정답 label을 기준으로 좋은 분기 조건을 학습하는 Supervised Learning 알고리즘이다.

# 2. Decision Tree의 Information Gain

## Information Gain 개념

Information Gain은 Decision Tree에서 어떤 Feature로 데이터를 나누었을 때 **불확실성 또는 impurity가 얼마나 줄어드는지**를 나타내는 값이다.

즉, 어떤 Feature로 나누면 데이터가 더 깔끔하게 분리되는지를 수치로 표현한 것이다.

Decision Tree는 보통 Information Gain이 큰 Feature를 먼저 선택한다.

## Entropy 개념

Entropy는 데이터가 얼마나 섞여 있는지를 나타내는 값이다.

| 데이터 상태 | Entropy 의미 |
|---|---|
| 한 class만 존재 | Entropy가 0, 완전히 순수함 |
| class가 반반 섞임 | Entropy가 최대 |
| 한쪽 class가 더 많음 | Entropy가 중간 수준 |

이진 분류에서 Entropy 공식은 다음과 같다.

$$
Entropy(S) = -p_+ \log_2(p_+) - p_- \log_2(p_-)
$$

| 기호 | 의미 |
|---|---|
| p+ | Positive class 비율 |
| p- | Negative class 비율 |

## Information Gain 공식

$$
Information\ Gain = Entropy(parent) - \sum_{i=1}^{k} \frac{|S_i|}{|S|}Entropy(S_i)
$$

말로 풀면 다음과 같다.

Information Gain은 부모 노드의 Entropy에서 자식 노드들의 가중 평균 Entropy를 뺀 값이다.

즉,

Information Gain = 나누기 전 불순도 - 나눈 후 불순도

## Information Gain 직관

전체 데이터가 다음과 같다고 하자.

| Class | 개수 |
|---|---:|
| Yes | 6 |
| No | 4 |
| 총합 | 10 |

이 데이터를 어떤 Feature로 나누었더니 다음처럼 되었다.

| 그룹 | Yes | No | 해석 |
|---|---:|---:|---|
| Group A | 5 | 0 | 완전히 순수함 |
| Group B | 1 | 4 | 대부분 No |

이 경우 나눈 뒤 class가 훨씬 덜 섞이므로 Information Gain이 크다.

반대로 다음과 같이 나누어지면 좋지 않다.

| 그룹 | Yes | No | 해석 |
|---|---:|---:|---|
| Group A | 3 | 2 | 여전히 섞임 |
| Group B | 3 | 2 | 여전히 섞임 |

나눈 뒤에도 불확실성이 거의 줄지 않았기 때문에 Information Gain이 작다.

# 3. K-means Clustering의 초기화 방식

## K-means Clustering 개념

K-means Clustering은 데이터를 K개의 cluster로 나누는 Unsupervised Learning 알고리즘이다.

기본 과정은 다음과 같다.

| 순서 | 과정 |
|---:|---|
| 1 | K개의 centroid를 초기화한다. |
| 2 | 각 데이터를 가장 가까운 centroid에 할당한다. |
| 3 | 각 cluster에 속한 데이터들의 평균으로 centroid를 갱신한다. |
| 4 | 데이터 할당과 centroid 갱신을 반복한다. |
| 5 | centroid 변화가 작아지면 종료한다. |

K-means에서 중요한 것은 초기 centroid를 어디에 두는가이다.

초기값이 좋지 않으면 local optimum에 빠지거나 수렴이 느려질 수 있다.

## Random Initialization

Random Initialization은 전체 데이터 중 K개의 sample을 무작위로 선택하고, 이들을 초기 centroid로 사용하는 방식이다.

예를 들어 K = 3이면 데이터 중 임의의 점 3개를 고른다.

## Random Initialization의 장점과 단점

| 구분 | 내용 |
|---|---|
| 장점 | 단순하고 빠르며 구현이 쉽다. |
| 단점 | 실행할 때마다 결과가 달라질 수 있다. |
| 단점 | 초기 centroid가 한쪽에 몰릴 수 있다. |
| 단점 | local optimum에 빠질 수 있다. |
| 단점 | outlier가 centroid로 선택되면 품질이 나빠질 수 있다. |

## Forgy Method

Forgy Method는 Random Initialization의 대표적인 방식이다.

데이터셋에서 K개의 실제 데이터 포인트를 랜덤하게 선택해서 초기 centroid로 사용한다.

보통 K-means의 랜덤 초기화라고 하면 Forgy Method를 의미하는 경우가 많다.

## Random Partition Method

Random Partition Method는 먼저 각 데이터를 무작위로 K개의 cluster 중 하나에 배정한다.

그다음 각 cluster의 평균을 계산해서 초기 centroid로 사용한다.

| 항목 | 설명 |
|---|---|
| 초기 centroid 위치 | 실제 데이터 포인트가 아닐 수 있다. |
| 장점 | 구현이 쉽다. |
| 단점 | 실제 cluster 구조와 무관하게 시작할 수 있다. |

## K-means++

K-means++는 Random Initialization의 단점을 줄이기 위한 대표적인 초기화 방식이다.

핵심은 centroid들이 서로 너무 가까이 몰리지 않도록 선택하는 것이다.

동작 방식은 다음과 같다.

| 순서 | 과정 |
|---:|---|
| 1 | 첫 번째 centroid를 랜덤하게 선택한다. |
| 2 | 각 데이터와 가장 가까운 centroid 사이의 거리를 계산한다. |
| 3 | 거리의 제곱에 비례하는 확률로 다음 centroid를 선택한다. |
| 4 | K개의 centroid가 선택될 때까지 반복한다. |

K-means++는 이미 선택된 centroid와 멀리 떨어진 데이터가 다음 centroid로 선택될 확률을 높인다.

## K-means++의 장점과 단점

| 구분 | 내용 |
|---|---|
| 장점 | centroid가 한쪽에 몰리는 문제를 줄인다. |
| 장점 | 수렴이 빠른 편이다. |
| 장점 | 결과가 Random Initialization보다 안정적이다. |
| 단점 | 초기화 과정에서 거리 계산 비용이 증가한다. |
| 단점 | 완전한 global optimum을 보장하지는 않는다. |
| 단점 | outlier가 선택될 가능성이 있다. |

## Maximin Initialization

Maximin Initialization은 centroid 사이의 거리를 최대한 멀게 선택하는 방식이다.

첫 번째 centroid는 랜덤하게 선택하고, 이후 centroid는 기존 centroid들과 가장 멀리 떨어진 점을 선택한다.

| 장점 | 단점 |
|---|---|
| centroid가 넓게 퍼진다. | outlier에 취약하다. |
| 구현이 직관적이다. | 데이터 밀도보다 거리만 반영할 수 있다. |

## PCA 기반 초기화

PCA 기반 초기화는 PCA(Principal Component Analysis)를 이용해서 데이터가 많이 퍼진 방향을 찾고, 그 방향을 따라 centroid를 배치하는 방식이다.

| 장점 | 단점 |
|---|---|
| 데이터의 분산 구조를 반영한다. | PCA 계산 비용이 필요하다. |
| 랜덤성이 줄어든다. | 비선형 cluster 구조에는 약할 수 있다. |

## Multiple Random Starts

Multiple Random Starts는 K-means를 여러 번 실행한 뒤 가장 좋은 결과를 선택하는 방식이다.

보통 WCSS(Within-Cluster Sum of Squares) 또는 inertia가 가장 작은 결과를 선택한다.

| 장점 | 단점 |
|---|---|
| 나쁜 초기값 하나에 의존하지 않는다. | 실행 횟수만큼 계산 비용이 증가한다. |
| 결과 안정성이 높아진다. | 완전한 global optimum 보장은 어렵다. |

## 초기화 방식 비교

| 초기화 방식 | 핵심 아이디어 | 장점 | 단점 |
|---|---|---|---|
| Forgy Method | 데이터 중 K개 랜덤 선택 | 단순하고 빠름 | 초기값에 민감 |
| Random Partition | 데이터를 랜덤 cluster에 배정 후 평균 계산 | 구현 쉬움 | 실제 구조 반영 약함 |
| K-means++ | 멀리 떨어진 점을 확률적으로 선택 | 안정적이고 수렴 빠름 | 초기화 비용 증가 |
| Maximin | 가장 멀리 떨어진 점 선택 | centroid 분산 좋음 | outlier에 취약 |
| PCA 기반 초기화 | 주성분 방향에 centroid 배치 | 분산 구조 반영 | 비선형 구조에 약함 |
| Multiple Random Starts | 여러 번 실행 후 최선 선택 | 안정성 높음 | 계산 비용 증가 |

실무에서는 보통 **K-means++와 Multiple Random Starts** 조합을 많이 사용한다.

# 4. Bayes' Theorem

## Bayes' Theorem 개념

Bayes' Theorem은 어떤 증거가 관측되었을 때 원인이 무엇일 가능성이 높은지를 계산하는 방법이다.

쉽게 말하면 새로운 증거를 보고 기존 확률을 업데이트하는 공식이다.

## Bayes' Theorem 공식

$$
P(A|B)=\frac{P(B|A)P(A)}{P(B)}
$$

각 항의 의미는 다음과 같다.

| 기호 | 의미 |
|---|---|
| P(A) | A가 원래 일어날 확률, Prior Probability |
| P(B) | B가 일어날 전체 확률, Evidence |
| P(A\|B) | B가 일어났을 때 A가 일어날 확률, Posterior Probability |
| P(B\|A) | A가 일어났을 때 B가 일어날 확률, Likelihood |

## 주요 용어

| 용어 | 의미 |
|---|---|
| Prior Probability | 증거를 보기 전의 확률 |
| Likelihood | 가설이 참일 때 증거가 관측될 확률 |
| Posterior Probability | 증거를 본 뒤 업데이트된 확률 |
| Evidence | 증거 자체가 나타날 전체 확률 |

즉, Bayes' Theorem은 다음 구조로 이해할 수 있다.

$$
Posterior = \frac{Likelihood \times Prior}{Evidence}
$$

## 질병 검사 예제

어떤 질병 검사에 대해 다음 정보가 있다고 하자.

| 항목 | 값 |
|---|---:|
| 질병 보유 확률 | 1% |
| 질병이 있을 때 양성 확률 | 99% |
| 질병이 없을 때 양성으로 잘못 나올 확률 | 5% |

검사 결과가 양성일 때 실제로 질병이 있을 확률을 구하라.

사건을 다음과 같이 정의한다.

| 사건 | 의미 |
|---|---|
| D | 실제로 질병이 있음 |
| + | 검사 결과가 양성 |

구하려는 값은 다음이다.

$$
P(D|+)
$$

Bayes' Theorem을 적용하면 다음과 같다.

$$
P(D|+) = \frac{P(+|D)P(D)}{P(+)}
$$

전체 양성 확률은 다음과 같다.

$$
P(+)=P(+|D)P(D)+P(+|\neg D)P(\neg D)
$$

값을 대입하면 다음과 같다.

$$
P(+)=0.99 \times 0.01 + 0.05 \times 0.99
$$

$$
P(+)=0.0099 + 0.0495 = 0.0594
$$

따라서

$$
P(D|+)=\frac{0.99 \times 0.01}{0.0594}
$$

$$
P(D|+)=\frac{0.0099}{0.0594}=0.1667
$$

정답은 약 **16.67%**이다.

## 숫자로 이해하기

전체 인구를 10,000명이라고 가정한다.

| 구분 | 계산 | 인원 |
|---|---|---:|
| 실제 질병 있음 | 10,000 × 1% | 100명 |
| 실제 질병 없음 | 10,000 × 99% | 9,900명 |

검사 결과는 다음과 같다.

| 구분 | 계산 | 인원 |
|---|---|---:|
| 질병 있음, 양성 | 100 × 99% | 99명 |
| 질병 있음, 음성 | 100 × 1% | 1명 |
| 질병 없음, 양성 | 9,900 × 5% | 495명 |
| 질병 없음, 음성 | 9,900 × 95% | 9,405명 |

양성으로 나온 사람은 99명 + 495명 = 594명이다.

그중 실제 질병이 있는 사람은 99명이다.

따라서

$$
P(D|+) = \frac{99}{594}=16.67\%
$$

## P(A|B)와 P(B|A)의 차이

P(A|B)와 P(B|A)는 서로 다르다.

| 확률 | 의미 |
|---|---|
| P(양성\|질병) | 질병이 있을 때 검사 결과가 양성일 확률 |
| P(질병\|양성) | 검사 결과가 양성일 때 실제로 질병이 있을 확률 |

검사 정확도가 높아 보여도 질병 자체가 드물면 P(질병|양성)은 생각보다 낮을 수 있다.

이를 Base Rate Problem이라고 한다.

# 5. Association Rule의 Support

## Association Rule 개념

Association Rule은 item 간의 동시 등장 관계를 찾는 방법이다.

대표적인 형태는 다음과 같다.

$$
A \rightarrow B
$$

이 의미는 A를 가진 transaction에서 B도 같이 나타나는 경향이 있다는 뜻이다.

예를 들어 우유를 산 사람이 빵도 같이 사는 경향이 있는지 분석할 수 있다.

## Support 개념

Support는 어떤 itemset이 전체 transaction 중에서 얼마나 자주 등장하는지를 나타낸다.

규칙 A → B의 Support는 A와 B가 동시에 등장한 transaction 비율이다.

$$
Support(A \rightarrow B) = Support(A \cup B)
$$

$$
Support(A \rightarrow B) = \frac{A와\ B를\ 모두\ 포함한\ transaction\ 수}{전체\ transaction\ 수}
$$

중요한 점은 Support는 A만 등장한 비율도 아니고, B만 등장한 비율도 아니라는 것이다.

반드시 A와 B가 동시에 등장한 비율이다.

## Support 예제

다음 장바구니 데이터가 있다.

| Transaction | 구매 item |
|---|---|
| T1 | 우유, 빵 |
| T2 | 우유, 기저귀, 맥주 |
| T3 | 빵, 기저귀 |
| T4 | 우유, 빵, 기저귀 |
| T5 | 우유, 빵, 맥주 |

규칙 우유 → 빵의 Support를 구해보자.

우유와 빵이 동시에 등장한 transaction은 다음과 같다.

| Transaction | 구매 item |
|---|---|
| T1 | 우유, 빵 |
| T4 | 우유, 빵, 기저귀 |
| T5 | 우유, 빵, 맥주 |

동시 등장 transaction 수는 3개이고, 전체 transaction 수는 5개이다.

따라서

$$
Support(우유 \rightarrow 빵)=\frac{3}{5}=0.6
$$

정답은 **60%**이다.

## 방향은 중요하지 않다

Support는 A와 B가 같이 등장한 비율이므로 방향이 바뀌어도 같다.

$$
Support(A \rightarrow B) = Support(B \rightarrow A)
$$

예를 들어 우유 → 빵과 빵 → 우유의 Support는 같다.

둘 다 우유와 빵이 동시에 등장한 transaction 비율을 계산하기 때문이다.

## Support와 Confidence 차이

| 지표 | 공식 | 의미 |
|---|---|---|
| Support | A와 B 동시 등장 수 / 전체 transaction 수 | 전체 데이터에서 규칙이 얼마나 자주 나타나는가 |
| Confidence | A와 B 동시 등장 수 / A 등장 수 | A가 있을 때 B도 있을 확률 |
| Lift | Confidence(A → B) / Support(B) | A와 B가 우연보다 얼마나 강하게 연결되는가 |

앞의 예제에서 우유 → 빵을 다시 보자.

| 항목 | 값 |
|---|---:|
| 전체 transaction 수 | 5 |
| 우유 등장 transaction 수 | 4 |
| 우유와 빵 동시 등장 transaction 수 | 3 |

Support는 다음과 같다.

$$
Support(우유 \rightarrow 빵)=\frac{3}{5}=0.6
$$

Confidence는 다음과 같다.

$$
Confidence(우유 \rightarrow 빵)=\frac{3}{4}=0.75
$$

Support와 Confidence는 계산 기준이 다르다.

## 문제 5

어떤 질병의 유병률은 2%이다. 질병이 있을 때 양성 확률은 90%, 질병이 없을 때 양성 확률은 10%이다. 검사 결과가 양성일 때 실제 질병이 있을 확률은?

풀이:

$$
P(D|+)=\frac{P(+|D)P(D)}{P(+|D)P(D)+P(+|\neg D)P(\neg D)}
$$

$$
P(D|+)=\frac{0.9 \times 0.02}{0.9 \times 0.02 + 0.1 \times 0.98}
$$

$$
P(D|+)=\frac{0.018}{0.116}=0.155
$$

정답: **15.5%**
